# Contributing

## Scope
This repository accepts changes through pull requests.

## Compliance Requirements
Before merge, contributions must include:
- Clear authorship in Git commits.
- No secrets or credentials in tracked files.
- Third-party dependencies compliant with allowed licenses.
- Updated compliance docs when legal/commercial impact exists.

## Coding and Review Rules
- Avoid modifying business logic when working on compliance-only tasks.
- Keep changes atomic and auditable.
- Include rationale in PR description.

## Commit and PR Guidelines
- Use descriptive commit messages.
- Reference impacted modules/apps.
- For dependency additions, include license evidence.

## AI-Assisted Development
AI tooling may be used as drafting assistance only.
Human contributors remain responsible for architecture, code decisions,
and final review.
